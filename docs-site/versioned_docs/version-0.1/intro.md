# プロジェクト概要

特許侵害調査を自動化するシステム **IP Rich Tools** のドキュメントサイトです。

## システム概要

IP Rich Toolsは、特許データの取得・正規化から侵害分析までを段階的に自動化するシステムです。
現在の主開発は Phase 2（FastAPI + SQLAlchemy + Next.js フロントエンド）です。

## フェーズ状況

| Phase | 状態 | 内容 |
|-------|------|------|
| Phase 1 | 完了（アーカイブ） | PoC（Next.js）は整理済み |
| Phase 2 | 進行中（主開発） | パイプラインA/B/CとAPI・フロント整備 |
| Phase 3 | 予定 | J-PlatPat連携、運用効率化 |
| Phase 4 | 予定 | 外部公開、課金・ユーザー管理 |

## 業務フローと自動化対象

![業務フロー図](./diagrams/current-workflow.svg)

### 自動化対象

| 対象 | 内容 | 実装状況 |
|-----|------|---------|
| **① 特許一覧作成** | 特許データの取得・正規化 | Phase 2 で基盤整備中 |
| **② 侵害調査** | LLMによる侵害可能性分析 | Phase 2 で実装中 |
| **③ 売上推定** | 被疑侵害製品の売上・収益性分析 | Phase 3 で対応予定 |

### 人で対応する業務

- 無効調査（特許の有効性確認）
- 訴訟手続き（法的対応）
- 総合判断（訴訟可否の最終決定）

## Phase 詳細

### 🚧 Phase 2 進行中: 業務利用可能性検証

:::info 目標
iprich内で業務利用可能性を検証できる状態にする
:::

**達成するとできること**
- 複数の特許を一括登録し、分析パイプラインを通せる
- 侵害分析ジョブの状態管理と結果取得ができる
- フロントエンドから結果確認ができる

### 📋 Phase 3: 業務運用の効率化

:::info 目標
iprich社内で日常業務として利用可能な状態にする
:::

**達成するとできること**
- J-PlatPatから特許情報を自動取得（手入力不要）
- 侵害調査結果を管理し、条件検索で絞り込み
- 気になる結果のみ侵害額推定を実行
- 調査結果をCSVでエクスポート

:::warning 体制構築が必要
Phase 3以降は機能開発のボリュームが増加するため、エンジニアリソースの増強を検討
:::

### 🌐 Phase 4: 外部公開（商用利用）

:::info 目標
外部顧客へのサービス提供と収益化
:::

**達成するとできること**
- 外部の企業・特許事務所にサービスとして提供
- ユーザーごとにログインし、自社データのみ閲覧
- ユーザーグループ（企業）単位で利用料を管理・請求

:::warning 体制構築が必要
商用サービスとして運用するため、開発・運用体制の本格的な構築が必要
:::

## システムの使い方

### Step 1: 特許データの取り込み

- CLIで特許データを取り込み（`python -m app.cli ingest`）
- 取り込み済みのデータをパース（`python -m app.cli parse`）

### Step 2: 侵害分析ジョブの開始

- API `POST /v1/analysis/start` でジョブ作成
- Cronでバッチ実行（`/api/cron/batch-analyze`）

### Step 3: 結果の確認

- API `GET /v1/analysis/{job_id}/results` で取得
- フロントエンドで一覧・詳細を確認
