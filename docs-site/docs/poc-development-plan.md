## Phase 1: 侵害調査の自動化（PoC版・Difyなし）

**期間**: 2〜3週間
**目標**: 特許1件あたりの侵害可能性を自動判定（Difyを使わず直接実装）

### 実装範囲

1. **構成要件抽出**
   - 入力: 特許番号 + 請求項1
   - 処理: Claude 3.5 Sonnetで構成要件を分解
   - 出力: 構造化された構成要件リスト

2. **製品検索・情報収集**
   - Tavily API無料枠を使用（1000検索/月）
   - または Claude DeepSearch（手動実行 → システム連携）
   - フォールバック: axios + cheerio（Node.js）

3. **充足性判定**
   - Claude 3.5 Sonnetで各構成要件の充足性を判定
   - 根拠URLを記録

4. **結果保存**
   - ローカルJSONファイル or Supabase無料枠
   - CSVエクスポート機能付き

### 技術スタック（PoC版・Difyなし）

```typescript
技術スタック:
├─ フロントエンド: Next.js 14 + shadcn/ui
├─ バックエンド: Next.js API Routes
├─ LLM: Claude 3.5 Sonnet（無料枠$5）
├─ Web検索:
│  ├─ Tavily API（無料枠: 1000検索/月）
│  └─ フォールバック: Claude DeepSearch（手動）
├─ データ保存: ローカルJSON / Supabase（無料枠500MB）
└─ デプロイ: Vercel（無料枠）
```

### 実装ステップ

| ステップ | 作業内容                         | 期間 | 成果物                             |
| -------- | -------------------------------- | ---- | ---------------------------------- |
| 1        | Next.jsプロジェクト初期化        | 1日  | プロジェクト構造、環境設定         |
| 2        | Claude APIプロンプト最適化       | 2日  | 構成要件抽出・充足性判定プロンプト |
| 3        | コアモジュール開発（TypeScript） | 5日  | 侵害調査自動化モジュール           |
| 4        | Next.js UI構築（shadcn/ui）      | 3日  | Webインターフェース                |
| 5        | テスト・検証                     | 2日  | 精度検証レポート                   |

### コスト試算（Phase 1 PoC）

```
Claude API（3.5 Sonnet）無料枠$5:
  - 構成要件抽出: 1,000 tokens/件 × 10件 = 10,000 tokens
  - 充足性判定: 2,000 tokens/件 × 10件 = 20,000 tokens
  - 合計: 30,000 tokens ≈ $0.09（無料枠内）

Tavily API: 無料枠1000検索/月
Vercel: 無料枠
Supabase: 無料枠500MB

合計コスト: $0（完全無料）
```

## Phase 2: 売上推定の自動化（PoC版・Difyなし）

**期間**: 2〜3週間
**目標**: 侵害可能性○の製品の売上を自動推定

### 実装範囲

1. **企業情報収集**
   - Wikipedia APIで企業基本情報を取得（無料）
   - Crunchbase無料検索で資金調達情報
   - 企業公式サイトをスクレイピング（axios + cheerio）

2. **売上推定**
   - Claude 3.5 Sonnetで公開情報から売上を推定
   - 市場規模データ（Statista無料版、業界レポート）
   - 競合比較で妥当性チェック

3. **収益ランク判定**
   - ルールベース（A〜Eランク自動判定）

### 技術スタック（PoC版・継続使用）

```typescript
技術スタック:
├─ 企業情報:
│  ├─ Wikipedia API（無料）
│  ├─ Crunchbase 無料検索
│  └─ Web scraping（axios + cheerio）
├─ 売上推定: Claude 3.5 Sonnet
├─ データ保存: Supabase（継続）
└─ 統合: Phase 1のNext.jsアプリに追加
```

### 実装ステップ

| ステップ | 作業内容               | 期間 | 成果物                             |
| -------- | ---------------------- | ---- | ---------------------------------- |
| 1        | 企業情報収集モジュール | 3日  | Wikipedia/Crunchbase連携スクリプト |
| 2        | 売上推定プロンプト開発 | 2日  | 売上推定プロンプト                 |
| 3        | 収益ランク判定ロジック | 2日  | ランク判定関数                     |
| 4        | UI統合・テスト         | 3日  | 統合版UI + 検証レポート            |

### コスト試算（Phase 2 PoC）

```
Claude API（継続使用・無料枠内）:
  - 売上推定: 3,000 tokens/件 × 5件 = 15,000 tokens
  - 合計: $0.045（無料枠内）

Wikipedia API: 無料
Crunchbase: 無料検索枠内

合計コスト: $0（無料枠継続）
```

## Phase 3: 特許一覧作成の自動化（PoC版）

**期間**: 1〜2週間
**目標**: 特許権者名から特許リストを自動生成

### 実装範囲

1. **特許情報取得**
   - J-PlatPat Web画面をスクレイピング
   - または J-PlatPat APIを調査・利用（無料）
   - 特許番号、名称、請求項1を抽出

2. **データ構造化**
   - CSVで出力（特許番号、名称、請求項1、出願日）

### 技術スタック（PoC版）

```typescript
技術スタック:
├─ 特許検索:
│  ├─ J-PlatPat API（無料、要調査）
│  └─ フォールバック: Playwright + スクレイピング
├─ 請求項抽出: Claude 3.5 Sonnet（構造化）
├─ データ保存: CSV出力 + Supabase
└─ 統合: Phase 1-2のNext.jsアプリと統合
```

### 実装ステップ

| ステップ | 作業内容                 | 期間 | 成果物                 |
| -------- | ------------------------ | ---- | ---------------------- |
| 1        | J-PlatPat API調査        | 2日  | API仕様ドキュメント    |
| 2        | 特許検索・抽出スクリプト | 3日  | 特許一覧取得スクリプト |
| 3        | Phase 1-2と統合          | 2日  | 完全自動化ワークフロー |
| 4        | テスト                   | 2日  | 検証レポート           |

### コスト試算（Phase 3 PoC）

```
J-PlatPat API: 無料
Claude API（継続使用・無料枠内）:
  - 請求項構造化: 1,500 tokens/件 × 20件 = 30,000 tokens
  - 合計: $0.09（無料枠内）

合計コスト: $0（無料枠継続）
```

## PoC検証指標

PoCの成功を評価するための指標：

### 定量指標

| 指標         | 目標値     | 測定方法          |
| ------------ | ---------- | ----------------- |
| **精度**     | 80%以上    | 人間の判断と比較  |
| **処理時間** | 5分/件以下 | 実測値            |
| **コスト**   | $3/月以下  | APIコスト集計     |
| **自動化率** | 70%以上    | 自動処理 / 全処理 |

### 定性指標

- ユーザー満足度（技術担当者、知財担当者）
- 分析結果の妥当性
- システムの使いやすさ
- 継続利用意向
