# PoC開発計画（コスト削減版）

PoCフェーズでは、開発コストを最小限に抑えながら概念実証を行います。

## コスト削減戦略

| 項目 | 本番想定 | PoC版（コスト削減） | 削減効果 |
|------|----------|---------------------|----------|
| **LLM** | GPT-4 | GPT-3.5-turbo / GPT-4o-mini | 90%削減 |
| **オーケストレーション** | Dify Cloud Pro | Dify Cloud 無料プラン | 100%削減 |
| **Web検索** | Google Search API | Perplexity API無料枠 / SerpAPI無料枠 | 100%削減 |
| **企業情報** | 有料API | Web scraping + 公開データ | 100%削減 |
| **データストレージ** | DynamoDB | ローカルJSON/CSV | 100%削減 |
| **月額コスト** | $265〜640 | **$10〜30** | **95%削減** |

## Phase 1: 侵害調査の自動化（PoC版）

**期間**: 2〜3週間
**目標**: 特許1件あたりの侵害可能性を自動判定

### 実装範囲

1. **構成要件抽出**
   - 入力: 特許番号 + 請求項1
   - 処理: GPT-3.5-turboで構成要件を分解
   - 出力: 構造化された構成要件リスト

2. **製品検索・情報収集**
   - Perplexity API無料枠を使用（100リクエスト/日）
   - または SerpAPI無料枠（100検索/月）
   - フォールバック: Pythonスクレイピング（requests + BeautifulSoup）

3. **充足性判定**
   - GPT-3.5-turboで各構成要件の充足性を判定
   - 根拠URLを記録

4. **結果保存**
   - ローカルJSONファイルに保存
   - 後でCSVエクスポート可能

### 技術スタック（PoC版）

```python
技術スタック:
├─ フロントエンド: Streamlit（無料、簡易UI）
├─ LLM: OpenAI GPT-3.5-turbo（$0.0015/1K tokens）
├─ Web検索:
│  ├─ Perplexity API（無料枠: 100req/日）
│  └─ フォールバック: requests + BeautifulSoup
├─ データ保存: ローカルJSON/CSV
└─ デプロイ: ローカル実行（インフラコスト$0）
```

### 実装ステップ

| ステップ | 作業内容 | 期間 | 成果物 |
|---------|----------|------|--------|
| 1 | プロンプト最適化 | 2日 | 構成要件抽出・充足性判定プロンプト |
| 2 | Pythonスクリプト開発 | 5日 | 侵害調査自動化スクリプト |
| 3 | Streamlit UI構築 | 3日 | 簡易Webインターフェース |
| 4 | テスト・検証 | 3日 | 精度検証レポート |

### コスト試算（Phase 1 PoC）

```
OpenAI API（GPT-3.5-turbo）:
  - 構成要件抽出: 500 tokens/件 × 10件 = 5,000 tokens
  - 充足性判定: 1,000 tokens/件 × 10件 = 10,000 tokens
  - 合計: 15,000 tokens ≈ $0.02

Perplexity API: 無料枠内

合計コスト: $0.02〜1/月（テスト段階）
```

## Phase 2: 売上推定の自動化（PoC版）

**期間**: 2〜3週間
**目標**: 侵害可能性○の製品の売上を自動推定

### 実装範囲

1. **企業情報収集**
   - Wikipedia APIで企業基本情報を取得（無料）
   - Crunchbase無料検索で資金調達情報
   - 企業公式サイトをスクレイピング

2. **売上推定**
   - GPT-3.5-turboで公開情報から売上を推定
   - 市場規模データ（Statista無料版、業界レポート）
   - 競合比較で妥当性チェック

3. **収益ランク判定**
   - ルールベース（A〜Eランク自動判定）

### 技術スタック（PoC版）

```python
技術スタック:
├─ 企業情報:
│  ├─ Wikipedia API（無料）
│  ├─ Crunchbase 無料検索
│  └─ Web scraping（requests + BeautifulSoup）
├─ 売上推定: GPT-3.5-turbo
├─ データ保存: ローカルJSON/CSV
└─ 統合: Phase 1のStreamlitに追加
```

### 実装ステップ

| ステップ | 作業内容 | 期間 | 成果物 |
|---------|----------|------|--------|
| 1 | 企業情報収集モジュール | 3日 | Wikipedia/Crunchbase連携スクリプト |
| 2 | 売上推定プロンプト開発 | 2日 | 売上推定プロンプト |
| 3 | 収益ランク判定ロジック | 2日 | ランク判定関数 |
| 4 | UI統合・テスト | 3日 | 統合版UI + 検証レポート |

### コスト試算（Phase 2 PoC）

```
OpenAI API（GPT-3.5-turbo）:
  - 売上推定: 2,000 tokens/件 × 5件 = 10,000 tokens
  - 合計: $0.015

Wikipedia API: 無料
Crunchbase: 無料検索枠内

合計コスト: $0.02〜1/月
```

## Phase 3: 特許一覧作成の自動化（PoC版）

**期間**: 1〜2週間
**目標**: 特許権者名から特許リストを自動生成

### 実装範囲

1. **特許情報取得**
   - J-PlatPat Web画面をスクレイピング
   - または J-PlatPat APIを調査・利用（無料）
   - 特許番号、名称、請求項1を抽出

2. **データ構造化**
   - CSVで出力（特許番号、名称、請求項1、出願日）

### 技術スタック（PoC版）

```python
技術スタック:
├─ 特許検索:
│  ├─ J-PlatPat API（無料、要調査）
│  └─ フォールバック: Selenium + スクレイピング
├─ 請求項抽出: GPT-3.5-turbo（構造化）
├─ データ保存: CSV出力
└─ 統合: Phase 1-2と統合
```

### 実装ステップ

| ステップ | 作業内容 | 期間 | 成果物 |
|---------|----------|------|--------|
| 1 | J-PlatPat API調査 | 2日 | API仕様ドキュメント |
| 2 | 特許検索・抽出スクリプト | 3日 | 特許一覧取得スクリプト |
| 3 | Phase 1-2と統合 | 2日 | 完全自動化ワークフロー |
| 4 | テスト | 2日 | 検証レポート |

### コスト試算（Phase 3 PoC）

```
J-PlatPat API: 無料
OpenAI API（GPT-3.5-turbo）:
  - 請求項構造化: 800 tokens/件 × 20件 = 16,000 tokens
  - 合計: $0.024

合計コスト: $0.03〜1/月
```

**目標**: 全プロセスの統合とダッシュボード表示

## PoC検証指標

PoCの成功を評価するための指標：

### 定量指標

| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| **精度** | 80%以上 | 人間の判断と比較 |
| **処理時間** | 5分/件以下 | 実測値 |
| **コスト** | $3/月以下 | APIコスト集計 |
| **自動化率** | 70%以上 | 自動処理 / 全処理 |

### 定性指標

- ユーザー満足度（技術担当者、知財担当者）
- 分析結果の妥当性
- システムの使いやすさ
- 継続利用意向
