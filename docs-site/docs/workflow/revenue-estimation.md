# 売上推定ワークフロー

被疑侵害製品の売上・収益性を推定するワークフローの詳細です。

## 現行（手動）

- 侵害可能性ありの場合のみ実行
- プロンプトを手動で作成・実行
- 売上情報を手動で収集

## 自動化後

- 侵害可能性○の場合、自動で起動
- 企業・製品情報を自動収集
- 財務情報から売上を推定
- 収益ランク（A〜E）を自動判定

## 収益ランク基準

| ランク | 売上規模 | 説明 |
|-------|---------|------|
| **Aランク** | 100億円以上 | 大手企業の主力製品、高優先度 |
| **Bランク** | 10億円〜100億円 | 中規模市場、検討価値あり |
| **Cランク** | 1億円〜10億円 | ニッチ市場向け |
| **Dランク** | 1,000万円〜1億円 | 特定用途向け製品 |
| **Eランク** | 1,000万円未満 | 小規模市場、優先度低 |

## 収益見込みランクの分布

分析対象特許における収益見込みの傾向：

| ランク | 割合 | 特徴 |
|-------|-----|------|
| Aランク | 約40% | 基本特許、大手IT企業が対象（スマートホーム、セキュリティ、AR/VR） |
| Bランク | 約10% | 中規模市場の特許 |
| Cランク | 約5% | ニッチ市場向け |
| Dランク | 約20% | 特定用途向け製品 |
| Eランク | 約5% | 小規模市場、研究開発段階 |
| 侵害可能性なし | 約20% | − |

## プロンプトテンプレート

```
以下の被疑侵害製品について、日本市場における売上を推定せよ。

▼対象製品情報：
・製品名: {製品名}
・企業名: {企業名}
・特許番号: {特許番号}
・侵害機能: {侵害が疑われる機能}

▼推定要件：
・日本市場での売上を推定すること
・推定根拠（企業財務情報、市場シェア、公開情報等）を明記すること
・複数の情報源から推定値の妥当性を検証すること
・推定の確実性レベル（高/中/低）を示すこと

▼出力フォーマット：
**1. 企業情報**
- 企業名:
- 本社所在地:
- 日本法人の有無:
- 主要事業:

**2. 製品情報**
- 製品名:
- カテゴリ:
- 主要機能:
- 価格帯:

**3. 市場情報**
- 市場規模（日本）:
- 市場成長率:
- 主要競合:
- シェア推定:

**4. 売上推定**
- 推定売上（年間）: XX億円
- 推定根拠:
  - [根拠1とURL]
  - [根拠2とURL]
  - [根拠3とURL]
- 推定の確実性: [高/中/低]
- 収益見込みランク: [A/B/C/D/E]

**5. 参考情報**
- 特許侵害部分の売上寄与率推定:
- その他の考慮事項:
```

## 売上推定の実例

### 例1: スマートホーム関連特許（特許05148670）

| 被疑侵害製品 | 推定売上 |
|------------|---------|
| Amazon Alexa | 約300億円 |
| Philips Hue | 約500億円 |
| SwitchBotシリーズ | 約200億円 |

**収益見込み**: Aランク（合計 約1,000億円）

### 例2: ネットワークセキュリティ特許（特許04895405）

| 被疑侵害製品 | 推定売上 |
|------------|---------|
| Cisco | 約35,000億円 |
| Palo Alto Networks | 約1,000億円 |
| Check Point Software | 約800億円 |
| Forescout | 約150億円 |
| Huawei | 約2,500億円 |

**収益見込み**: Aランク（合計 約39,450億円）

## 推定時の注意点

### データの正確性

- 公式ソースの優先
- 情報の鮮度確認
- 複数ソースでのクロスチェック

### 推定の限界

- 非公開企業の財務情報は推定に頼る
- 地域別売上の内訳が不明な場合がある
- 特許侵害部分の売上寄与率は推定値
