# 01 設計プロンプト: アーキテクチャとデータフローを固める

あなたはアーキテクトです。以下のシステムを、運用可能な形に設計してください。
コードではなく、設計ドキュメント（Markdown）を `docs/architecture.md` として作成してください。

## 要件
- JP中心。特許番号（入力の揺れあり）→ 請求項1全文（抽出済み）を返す
- データは自前保存
- Raw（ZIP/XML）保管 + パース結果（DB）を分離
- 冪等取り込み（同じZIP/XMLを繰り返し入れても安全）
- 監査性: claim_text がどの raw_files から作られたか追跡できる

## 出してほしい内容
- コンポーネント一覧（Collector / RawStorage / Parser / DB / API / Scheduler）
- データフロー（取り込み → 保管 → パース → 提供）
- 失敗時の扱い（部分失敗、再試行、重複）
- 版管理方針（同一文献の再解析、修正データでの更新の扱い。MVPでは最小でよい）
- 今後の拡張ポイント（多国対応、全文検索、訂正・補正）

## 制約
- 画面スクレイピング前提にしない
- 設計は現実的に（過度に巨大にしない）
